Basic Usage

The usage of the Tableau Prover is kept relatively simple. Formulae are entered in the text areas in the center of the drawing canvas. To enter multiple formulae, enter those in different lines. To start the atomatic or manual calculation, press the "Calculate"-button below the text areas. The Settings are described in the settings section.

Settings

There are two kinds of settings. The first one defines the logic used and the second one whether or not the Tableau should be automatically solved. The available logic types are "Propositional Logic (PL)", "First-Order Predicate Logic (FOPL)", "Intuitionistic Propositional Logic (IPL)" and "Intuitionistic First-Order Predicate Logic (IFOPL)" To change the logic, change the selected radio button on top of the window.

To change, whether or not the Tableau should be solved automatically, change the selected radio button below the logic settings. If "Manual" is selected, you can enter the steps of solving the Tableau manually. The program checks, whether or not the entered steps are valid. If "Automatic" is selected, the Tableau is fully solved automatically if possible.

After calculating a Tableau, it is possible to reset the screen by pressing the X-Button at the upper left corner of the Scroll-Area.

Propositional Logic

If the logic type "Propositional Logic (PL)" is selected, the syntax and operator precedences defined in this section are applied.

Syntax Definition

The Propositional Logic has 5 Operators (Negation, Conjunction, Disjunction, Conditional, and Biconditional). These can be entered as Keyboard-Friendly characters:

  • Negation (¬): ! or -
  • Conjunction (∧): & or *
  • Disjunction (∨): | or +
  • Conditional (→): ->
  • Biconditional (↔): <->

Operator Precedences

To avoid unreadability due to too many clamps, the Operators got Precedences. An Operator with higher Precedence is evaluated first. I.e. with & having higher precedence then |, (A&B)|C is the same as A&B|C. The Operator Precedences are as follows:

  • 1. Negation
  • 2. Conjunction
  • 3. Disjunction
  • 4. Conditional
  • 5. Biconditional

First-Order Predicate Logic

If the logic type "First-Order Predicate Logic (FOPL)" is selected, the syntax and operator precedences defined in this section are applied. Please note that it is possible to enter functions, for example in P(f(X, Y)) the Predicate P got the function f as an argument. But there is no possibility to enter a custom evaluation of functions. Functions are considered equal, if and only if, all its input arguments are equal.

Syntax Definition

First-Order Predicate Logic has 7 Operators (Negation, Conjunction, Disjunction, Conditional, Biconditional, Univeral Quantification, and Existential Quantification). These can be entered as Keyboard-Friendly characters:

  • Universal Quantifier (∀): (A) or /\
  • Existential Quantifier (∃): (E) or \/
  • Negation (¬): ! or -
  • Conjunction (∧): & or *
  • Disjunction (∨): | or +
  • Conditional (→): ->
  • Biconditional (↔): <->
Constants, Predicates have to start with an uppercase letter.
Variables, Functions have to start with a lowercase letter.

A quantifier is always followed by one ore more comma separated variable names, followed by a space, followed by the expression in the scope of the quantifier. Examples are:
  • (A)x P(x)
  • (E)x P(x)
  • (A)x,y (P(x)&P(y))

Operator Precedences

To avoid unreadability due to too many clamps, the Operators got Precedences. An Operator with higher Precedence is evaluated first. I.e. with & having higher precedence then |, (A&B)|C is the same as A&B|C. The Operator Precedences are as follows:

  • 1. Universal Quantification
  • 2. Existential Quantification
  • 3. Negation
  • 4. Conjunction
  • 5. Disjunction
  • 6. Conditional
  • 7. Biconditional
